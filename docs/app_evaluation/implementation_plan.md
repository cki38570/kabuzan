# アプリ評価とAI分析の信頼性調査計画

この計画では、現在のシステムのコードベースを詳細に調査し、AIによる株価分析がどの程度信頼できるか、またその正確性を支える仕組みについて客観的なレポートを作成します。

## 調査のポイント
1. **プロンプトの設計思想**: 単なる予測ではなく「投資委員会」形式を採用していることの利点。
2. **データの裏付け**: テクニカル指標、マクロ環境、需給（信用残）、ニュース、決算データがどのように統合されているか。
3. **正確性の限界**: データ遅延やAIのハルシネーション（平気で嘘をつく可能性）に対するガードレール。
4. **過去の学習**: バックテスト結果や過去の分析履歴をどう活用しているか。

## 検討事項
- 現在のプロンプトで「定量的根拠」を強制している点は高く評価できますが、実際の計算値がAIに渡される際の誤差がないか確認します。
- 日本株特有の要素（主要株主の動き、空売り残高など）の考慮漏れがないか。

## 検証方法
- `analysis.py` でのシグナル判定ロジックを読み取り、AIに渡される情報の精度を評価。
- `llm.py` の `_create_mock_report` (エラー時のフォールバック) がどの程度妥当か。
